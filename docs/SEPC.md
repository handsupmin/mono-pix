# MonoPix Spec

## 1. 개요

**MonoPix**는 사용자가 업로드한 이미지를 픽셀 아트 스타일로 변환하는 웹 기반 도구이다.

- 이름: **MonoPix**
- README 첫줄: **MonoPix (Monocle + Pixel)**
- tagline: **See the world through a pixel monocle.**
- description: **Convert any image into pixel art. As if you're looking through a pixelated lens.**

MonoPix의 핵심 목표는 다음과 같다.

1. 사용자가 이미지를 간편하게 업로드할 수 있어야 한다.
2. 사용자가 직접 1:1 비율로 이미지를 crop할 수 있어야 한다.
3. 선택한 해상도 기준으로 이미지를 픽셀화할 수 있어야 한다.
4. 변환 전/후를 쉽게 비교할 수 있어야 한다.
5. 결과 이미지를 다운로드할 수 있고, 이전 결과들이 로컬 브라우저에 저장될 수 있어야 한다.
6. 서버 없이 브라우저에서 동작하는 정적 웹앱을 우선 목표로 한다.

## 6. 전체 사용자 흐름

1. 사용자가 이미지를 업로드한다.
2. 좌측 메인 화면에서 이미지를 1:1 비율로 crop한다.
3. 우측 조작 메뉴에서 해상도 및 출력 크기 옵션을 선택한다.
4. 사용자가 변환 버튼을 누른다.
5. 앱이 픽셀화를 수행한다.
6. 로딩 오버레이와 진행률 표시가 노출된다.
7. 변환이 완료되면 좌측 메인 화면은 preview 전용으로 전환된다.
8. 사용자는 before / after / split 보기 모드를 선택해 결과를 확인한다.
9. 사용자는 결과 이미지를 다운로드할 수 있다.
10. 결과는 로컬 히스토리에 저장된다.

## 8. 좌측 메인 화면 동작

## 8.1 변환 전 상태

좌측 메인 화면은 **crop 편집 전용**이다.

포함 요소:

- preview placeholder 표시
- 업로드 버튼 표시
- drag & drop 업로드 지원
- 이미지 업로드 후 이미지 이동 가능
- 이미지 업로드 후 이미지 줌 가능
- 1:1 고정 비율 crop 가능
- 사용자는 필요 시 현재 선택 해상도 기준의 격자선을 overlay로 표시할 수 있어야 함

### 변환 전 요구사항

- crop 비율은 항상 **1:1 고정**
- 사용자는 이미지를 드래그하여 위치를 조정할 수 있어야 함
- 사용자는 줌으로 확대/축소할 수 있어야 함
- 사용자는 crop 영역에 맞게 이미지를 즉석에서 편집할 수 있어야 함
- 변환 버튼 누르기 전에는 결과 미리보기를 보여주지 않음

### Crop 초기값

이미지 업로드 직후 crop 박스의 초기 상태:

- 이미지의 가로/세로 중 작은 값을 `minDim`으로 계산
- `minDim >= 900`이면 crop 박스 크기: **900x900**
- `minDim < 900`이면 crop 박스 크기: **minDim × minDim**
- crop 박스 초기 위치: **이미지 정중앙**

---

## 8.2 변환 후 상태

좌측 메인 화면은 **preview 전용**이다.

변환 후에는 조작 메뉴에서 아래 보기 모드를 선택할 수 있어야 한다.

### 보기 모드

- 변환 전
- 변환 후
- 세로 split
- 가로 split

### 보기 모드 설명

- **변환 전**: crop된 원본 이미지를 보여줌
- **변환 후**: 픽셀화된 결과만 보여줌
- **세로 split**: 좌우 기준으로 변환 전/후 비교
- **가로 split**: 상하 기준으로 변환 전/후 비교

- 세로 split과 가로 split은 50:50 고정 비율로 표시

## 10. 조작 메뉴

우측 조작 메뉴에는 아래 항목이 포함된다.

### 10.1 해상도 설정

선택 가능한 해상도:

- 8x8
- 16x16
- 32x32 (default)
- 64x64
- 128x128
- 256x256

### 10.2 출력 크기 방식

아래 두 가지 중 하나를 선택할 수 있어야 한다.

#### A. crop한 이미지 사이즈대로 픽셀화

- 결과 이미지는 crop된 영역 크기를 유지한다.
- 단, 내부적으로는 선택한 해상도 단위로 평균 색을 계산한다.
- 예: crop 결과가 160x160이고, 해상도가 16x16이면, 각 셀은 10x10 크기로 렌더링된다.

#### B. 선택 해상도로 실제 축소

- 결과 이미지를 실제 선택 해상도 크기로 생성한다.
- 예: 16x16 선택 시 결과 파일은 실제 16x16 PNG가 된다.

### 10.3 완성 이미지 크기 미리보기

조작 메뉴에는 현재 설정 기준의 출력 이미지 크기를 표시해야 한다.

예:

- EN
  - Crop size: 160 x 160
  - Output mode: Keep cropped size
  - Final output: 160 x 160

- KR
  - Crop 크기: 160 x 160
  - 출력 방식: crop한 이미지 사이즈대로 픽셀화
  - 최종 출력 크기: 160 x 160

### 10.4 보기 모드 선택

변환 후에만 활성화된다.

옵션:

- 변환 전
- 변환 후
- 세로 split
- 가로 split

### 10.5 액션 버튼

- 변환 버튼
- 다운로드 버튼
- 설정 리셋 버튼

### 10.6 격자선 표시 (Grid Overlay)

crop 편집 화면에는 현재 선택한 해상도를 기준으로 한 격자선(grid overlay)을 표시할 수 있어야 한다.

#### 목적

- 사용자가 현재 선택한 해상도 기준으로 이미지가 어떻게 분할될지 직관적으로 이해할 수 있어야 한다.
- crop 위치를 조정할 때 픽셀 셀 단위를 예측할 수 있어야 한다.

#### 동작 방식

- 격자선 표시는 **토글 방식**으로 제공한다.
- 사용자는 우측 조작 메뉴에서 격자선 표시 여부를 직접 켜고 끌 수 있어야 한다.
- 격자선은 **변환 전 crop 편집 상태에서만 표시**된다.
- 변환 후 preview 상태에서는 격자선을 표시하지 않는다.

#### 기본값

- 기본값은 **OFF**로 한다.

#### 표시 기준

- 격자선은 현재 선택한 해상도 `N x N` 기준으로 crop 영역 위에 표시된다.
- 예:
  - 16x16 선택 시 crop 영역은 16 x 16 셀 기준 격자선으로 나뉘어 보인다.
  - 32x32 선택 시 crop 영역은 32 x 32 셀 기준 격자선으로 나뉘어 보인다.

#### 고해상도 처리

- 128x128, 256x256처럼 해상도가 높은 경우 전체 세부 격자를 모두 강하게 표시하면 가독성이 떨어질 수 있다.
- 따라서 고해상도에서는 시인성을 위해 다음 중 하나를 적용할 수 있다.
  - 격자선을 더 연하게 표시
  - 전체 세부 격자 대신 주요 구획선만 표시
- 이 경우에도 기준은 현재 선택한 해상도여야 한다.

#### 구현 원칙

- 격자선은 이미지 편집을 방해하지 않아야 한다.
- 격자선은 crop 대상 이미지를 가리지 않도록 얇고 단순한 overlay로 표현한다.
- 격자선은 이미지 데이터 자체에 포함되지 않으며, 편집 화면에서만 보이는 UI overlay이다.
- 다운로드 결과물에는 격자선이 포함되지 않는다.

## 11. 해상도 유효성 검사

해상도 옵션은 현재 crop 결과의 실제 출력 픽셀 크기 기준으로 검사해야 한다.

### 규칙

- 사용자는 현재 crop 크기보다 큰 해상도를 선택할 수 없다.
- 예: crop 결과가 31x31이면 32x32 이상 선택 불가
- crop 결과가 100x100이면 128x128, 256x256 선택 불가

### 동작 방식

- 유효하지 않은 해상도 옵션은 선택 불가 상태로 표시
- 이미 선택된 해상도가 crop 변경 후 유효하지 않게 되면:
  - 자동으로 가장 가까운 유효 해상도로 바꿈

## 12. 핵심 픽셀화 로직

MonoPix의 핵심 로직은 **셀 단위 대표 색 계산**이다.

### Convert Type

사용자는 셀의 대표 색을 계산하는 방식을 두 가지 중 선택할 수 있다.

| 타입  | 이름                    | 설명                                                      |
| ----- | ----------------------- | --------------------------------------------------------- |
| **1** | 최빈 색 (Most Frequent) | 셀 내 픽셀 중 가장 많이 등장한 색 (quantize 후 빈도 기준) |
| **2** | 평균 색 (Average)       | 셀 내 픽셀의 RGB 평균값                                   |

- 기본값: **1 (최빈 색)**
- 최빈 색은 색을 8단계로 quantize한 뒤 빈도를 계산한다.

### 처리 순서

1. 사용자가 1:1 crop 영역을 조정한다.
2. crop된 이미지를 기준으로 정방형 이미지를 얻는다.
3. 선택한 해상도 `N x N`에 맞게 이미지를 `N x N` 셀로 분할한다.
4. 선택한 Convert Type에 따라 각 셀의 대표 색을 계산한다.
5. 계산된 대표 색으로 해당 셀 전체를 채운다.
6. 선택한 출력 크기 방식에 따라 최종 이미지를 생성한다.

### 예시

- crop 결과: 48 x 48
- 선택 해상도: 16 x 16

이 경우:

- 전체 이미지는 16 x 16 셀로 나뉜다.
- 각 셀은 3 x 3 영역을 담당한다.
- 각 3 x 3 영역에서 Convert Type에 따라 대표 색을 계산한다.
- 결과는 16 x 16개의 셀로 구성된 픽셀 이미지가 된다.

#### 출력 방식 A: crop한 이미지 사이즈대로 픽셀화

- 최종 출력 크기: 48 x 48
- 각 셀은 3 x 3의 동일한 색 블록으로 렌더링된다

#### 출력 방식 B: 선택 해상도로 실제 축소

- 최종 출력 크기: 16 x 16
- 각 셀은 실제 1픽셀로 저장된다

### 셀 분할 보충 규칙

- crop 결과 크기가 선택 해상도로 정수 나눗셈이 되지 않아도 변환 가능해야 한다.
- 시스템은 crop 이미지를 선택 해상도 기준의 샘플링 그리드로 매핑하여 각 셀의 대표 색을 계산한다.

## 13. 알파 채널 처리

PNG 출력 시 알파 채널을 지원해야 한다.

### 규칙

- 각 셀 내부 픽셀 중 50% 초과가 투명이면 해당 셀은 투명 처리한다.
- 그렇지 않으면 보이는 픽셀들의 평균 색을 계산해 해당 셀을 채운다.
- 출력 포맷은 PNG로 하며, 투명한 셀은 투명하게 유지한다.

### 참고

- 출력 포맷이 PNG일 경우 알파 채널 유지
- JPG/JPEG 업로드라도 출력은 PNG로 통일

## 14. 변환 실행 방식

### 14.1 실행 방식

- 변환은 **버튼 클릭 방식**
- 이 외 트리거 없음

### 14.2 변환 버튼 활성화 조건

다음 조건이 모두 만족될 때만 활성화된다.

- 이미지가 업로드되어 있음
- crop 상태가 유효함
- 해상도 선택이 유효함
- 현재 변환 중이 아님

### 14.3 변환 전 상태

- 결과 preview 없음
- 다운로드 버튼 비활성화

### 14.4 변환 후 상태

- 결과 preview 활성화
- 다운로드 버튼 활성화
- 보기 모드 선택 활성화

## 15. 로딩 및 진행 상태

변환 처리 중에는 전체 화면 오버레이를 표시한다.

### 요구사항

- 화면 전체를 불투명한 레이어로 덮는다
- 중앙 또는 명확한 위치에 progress bar를 표시한다
- 현재 진행 상황 문구를 함께 표시한다

### 상태 문구 예시

- Preparing crop...
- Sampling colors...
- Rendering pixel image...
- Finalizing preview...

### 진행 상태 표시 목적

- 사용자가 현재 처리 단계와 진행 정도를 알 수 있어야 한다
- 처리 중 중복 클릭을 방지해야 한다

---

## 16. 다운로드

### 16.1 다운로드 포맷

- PNG

### 16.2 다운로드 가능 시점

- 변환 완료 후에만 가능

### 16.3 파일명 규칙

파일명은 원본 파일명을 기반으로 생성한다.

형식:
`<original-name>_<resolution>_<mode>.png`

예:

- `cat_32x32_original-size.png`
- `cat_16x16_resized.png`

여기서:

- `<resolution>`: 선택한 해상도
- `<mode>`:
  - `original-size`: crop한 이미지 사이즈대로 픽셀화
  - `resized`: 선택 해상도로 실제 축소

## 17. Reset 및 삭제 UX

### 17.1 설정 리셋 버튼

- 우측 조작 메뉴에 설정 리셋 버튼 제공
- 리셋 시 기본값:
  - 해상도: 32x32
  - 출력 크기 방식: crop한 이미지 사이즈대로 픽셀화
  - 보기 모드: 변환 전
- 이미지 자체는 유지하고 설정만 초기화하는 방식 권장

### 17.2 휴지통 아이콘

좌측 화면에 각 상태별 휴지통 아이콘을 둘 수 있다.

- 변환 전 화면: 업로드된 원본 이미지 제거
- 변환 후 화면: 현재 preview 중인 결과 이미지를 제거하고, 편집 모드로 돌아간다.
- 해당 결과가 히스토리에 저장되어 있었다면 히스토리에서도 함께 제거할 수 있다.

## 18. 히스토리 저장

변환된 결과는 로컬에 저장되어야 한다.

### 18.1 저장 목적

- 사용자가 이전 결과를 다시 확인할 수 있어야 한다
- 서버 없이 브라우저 단에서 이력을 유지한다

### 18.2 저장 위치

- 히스토리 메타데이터 및 결과 이미지 저장은 IndexedDB를 우선 사용한다.
- 단순 설정값 및 가벼운 UI 상태 저장은 localStorage를 사용할 수 있다.

### 18.3 저장 항목

각 히스토리 항목은 최소 아래 정보를 포함한다.

- 원본 파일명
- 생성 시각
- 선택 해상도
- 출력 모드
- 결과 이미지 데이터
- 썸네일 또는 미리보기 이미지

### 18.4 저장 개수 제한

- 최근 10개까지만 저장
- 초과 시 가장 오래된 항목부터 제거

### 18.5 히스토리 UI

- 결과 페이지에서 이전 결과가 누적되어 보여야 한다
- 각 항목은 다시 확인하거나 다운로드, 삭제할 수 있어야 한다
- 히스토리 목록은 **좌측 preview 화면 하단에 가로 스크롤 리스트(horizontal list)** 형태로 표시한다.
- 히스토리 패널은 숨기기/펼치기 토글 가능 (기본값: 펼쳐진 상태)
- hover 시 삭제 아이콘 표시
- 히스토리 전체 삭제 버튼 표시 (클릭 시 alert 표시 후 삭제)
- 히스토리에서 이전 결과를 선택하면 해당 결과를 preview 화면에서 확인할 수 있다.
- 히스토리 화면은 결과 확인 / 다운로드 / 삭제 전용으로 동작한다.

## 20. 에러 처리

### 20.1 업로드 에러

- 지원하지 않는 파일 형식
- 파일 크기 초과
- 이미지 크기 부족
- 파일 읽기 실패

### 20.2 편집/변환 에러

- 유효하지 않은 crop 상태
- 현재 crop 크기로 선택한 해상도를 처리할 수 없는 경우
- Canvas 렌더링 실패
- 다운로드 생성 실패

### 20.3 에러 메시지 원칙

- 사용자에게 왜 실패했는지 명확히 알려야 한다
- 가능하면 해결 방법을 같이 안내해야 한다
- 현재 선택 언어에 맞는 메시지 표시

예:

- "This file format is not supported."
- "Maximum upload size is 20MB."
- "Image must be at least 8x8."
- "Selected resolution is larger than the current crop size."

## 25. 현재 버전 요약

MonoPix 현재 버전은 아래 핵심 경험에 집중한다.

1. 이미지를 업로드한다
2. 좌측에서 1:1 crop을 직접 조정한다
3. 우측에서 해상도와 출력 방식 선택
4. 변환 버튼 클릭
5. 평균 색 기반 픽셀화 수행
6. 변환 후 결과 확인
7. before / after / split 비교
8. PNG 다운로드
9. 결과 로컬 저장

이 버전은 서버 없이 브라우저 단에서 동작하는 것을 목표로 한다.
